# 新闻抓取应用 - 需求文档

## 一、项目概述
一个基于Python的新闻抓取应用，支持从5大权威网站自动抓取新闻，通过Web界面查看和管理。

## 二、技术栈
| 组件 | 选型 | 说明 |
|------|------|------|
| 后端框架 | FastAPI | 现代高性能，异步友好 |
| 数据库 | SQLite | 轻量，无需额外安装 |
| 模板引擎 | Jinja2 | FastAPI自带 |
| 定时任务 | APScheduler | Python定时任务库 |
| 反爬虫方案 | Playwright | 轻量自动化浏览器 + 请求头伪装 + 延时 |
| 前端 | HTML + CSS + JavaScript | 配合Jinja2模板 |

## 三、数据来源（6大网站）
1. 财联社
2. 新华网
3. 东方财富
4. 36氪
5. 巨潮资讯（A股公告）
6. 纽约时报（The New York Times）

## 四、数据存储字段
每条新闻包含：
- `id` - 主键，自增
- `title` - 标题
- `content` - 正文内容
- `source` - 来源网站
- `publish_time` - 发布时间
- `url` - 原文链接（用于去重）
- `author` - 作者
- `summary` - 摘要
- `image_path` - 图片本地存储路径
- `news_type` - 新闻类型（纽约时报专用：wire/topstories）
- `created_at` - 抓取时间
- `updated_at` - 更新时间

## 五、抓取策略
### 5.1 抓取方式
- **定时抓取**：每6小时自动抓取一次
- **手动抓取**：Web界面提供手动触发按钮

### 5.2 抓取规则
- **数量限制**：每个网站最多抓取10条新闻
- **时间范围**：只抓取最近24小时内的新闻
- **去重机制**：根据原文链接（url）去重，已存在的新闻不重复保存
- **图片处理**：下载图片保存到本地，记录本地路径

### 5.3 纽约时报抓取策略
- **API接口**：
  - Wire API（实时快讯）：`https://api.nytimes.com/svc/news/v3/content/all/all.json`
  - Top Stories API（精选）：`https://api.nytimes.com/svc/topstories/v2/home.json`
- **数量限制**：每个接口最多抓取6条
- **内容分开**：
  - Wire API → 标记为 `news_type=wire` → 推送到"纽约时报最新资讯"
  - Top Stories API → 标记为 `news_type=topstories` → 推送到"纽约时报精选"
- **推送规则**：分开推送，每个类别推送5条
- **时间显示**：保留原始新闻时间（含时区）

### 5.4 反爬虫策略
- 请求头伪装（User-Agent、Referer等）
- 随机延时（2-5秒）
- Playwright浏览器自动化（应对动态加载页面）

## 六、筛选功能
### 6.1 筛选规则配置
- **包含关键词**：支持多个关键词，用逗号分隔
  - 匹配范围：标题 + 正文
  - 逻辑：OR（任意一个关键词命中即中）
- **排除关键词**：支持多个关键词，用逗号分隔
  - 匹配范围：标题 + 正文
  - 逻辑：OR（任意一个关键词命中即过滤）

### 6.2 筛选规则存储
- 配置保存到数据库或配置文件
- 支持动态修改，无需重启应用

## 七、Web界面功能
### 7.1 页面列表
1. **首页/新闻列表页**
   - 分页显示新闻列表
   - 显示标题、来源、发布时间、摘要
   - 支持按时间倒序排列
   - 筛选功能入口

2. **新闻详情页**
   - 展示完整新闻内容
   - 显示图片
   - 提供原文链接跳转

3. **抓取控制页**
   - 手动触发抓取按钮
   - 显示抓取状态（进行中/完成）
   - 显示最近一次抓取时间

4. **筛选规则管理页**
   - 配置包含关键词
   - 配置排除关键词
   - 保存/重置规则

5. **抓取日志页**
   - 显示抓取历史记录
   - 每次抓取的时间、抓取数量、成功/失败状态
   - 错误日志展示

## 八、预留接口（后续扩展）
### 8.1 飞书推送接口
- 接口名称：`/api/push/feishu`
- 功能：将筛选后的新闻推送到飞书群/机器人
- 参数：新闻ID列表、飞书Webhook URL
- 返回：推送结果

### 8.2 AI打分接口
- 接口名称：`/api/ai/score`
- 功能：接入大模型对新闻按规则打分
- 参数：新闻ID列表、评分规则配置
- 返回：每条新闻的分数及评分理由

### 8.3 投资决策接口
- 接口名称：`/api/ai/investment`
- 功能：基于AI分析生成投资决策建议
- 参数：新闻ID列表、投资偏好配置
- 返回：投资决策建议（买入/卖出/观望、理由）
